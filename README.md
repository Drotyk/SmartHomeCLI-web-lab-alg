# Smart Home Control System
Симуляція реалізація системи управління розумним будинком на основі архітектури 
**Клієнт-Сервер** Проект демонструє глибоке розуміння **ООП**, принципів **SOLID**, патернів проектування та модульної розробки на C++

# Опис проекту
Система розділена на два незалежних модулі:
- **Backend (C++)**: Високопродуктивний сервер, що працює з TCP сокетами. Реалізує бізнес-логіку, керує пристроями та забезпечує збереження стану (Persistence) у файлову базу даних. Код структурований на заголовні файли (.hpp) та файли реалізації (.cpp).
- **Frontend (PHP + CSS)**: Веб-інтерфейс у стилі Monochrome (Minimalism). Відповідає за візуалізацію кімнат, відправку команд та адаптивне відображення елементів керування.

# Функціонал та Можливості
- **Persistence (Збереження даних)**: Система автоматично зберігає стан усіх пристроїв у файл. Після перезапуску сервера всі налаштування (кімнати, температура, світло) відновлюються.
- **Групування**: Автоматичне об'єднання пристроїв у візуальні блоки по кімнатах.
- **Підтримка пристроїв**:
- **Light: Вкл/Викл**.
- **Dimmer:** Регулювання яскравості (0-100%).
- **AC (Conditioner)**: Регулювання температури (16-30°C), режими Cool/Off.
- **Fan**: Вентиляція.
- **Інтерфейс**: Стильний монохромний дизайн з окремим CSS файлом
  # Архітектура та Патерни (Design Patterns)
 Проект розроблено з дотриманням принципів SOLID:
-  **Factory Method (Фабричний метод)**:
  Реалізовано в DeviceFactory. Дозволяє створювати об'єкти (SimpleLight, AirConditioner) за їх текстовим ідентифікатором, отриманим по мережі. Це забезпечує легке розширення системи новими типами пристроїв.
- **Repository (Репозиторій)**:
  Клас SmartHomeHub виступає сховищем об'єктів. Він інкапсулює логіку додавання, видалення, пошуку та серіалізації даних у файл.
-  **Interface Segregation (ISP) & Polymorphism**:
  Усі пристрої наслідують абстрактний клас IDevice. Сервер працює з вказівниками std::shared_ptr<IDevice>, не знаючи деталей реалізації конкретного пристрою.
-  **Separation of Concerns**:
  Мережева логіка відокремлена від бізнес-логіки. Інтерфейс (Frontend) відокремлений від даних (Backend).
#  Інструкція із запуску1.
**1. Підготовка Backend (C++)** 
Відкрийте термінал у корені проекту:Bashcd backend
```#bash
mkdir -p database  # Створення папки для БД (якщо немає)
make               # Компіляція проекту через Makefile
```
Після успішної компіляції запустіть сервер:
```#bash
./server
```
Вивід: Smart Home Server (Structured) running on 8080...
**2. Запуск Frontend (PHP)** 
Відкрийте новий термінал (не закриваючи сервер):
```#bash
php -S localhost:8000
```
**3. Використання**
Відкрийте браузер та перейдіть за посиланням: http://localhost:8000

### Протокол комунікації

Обмін даними відбувається по протоколу TCP (порт 8080) у текстовому форматі з роздільником `|`.

| Дія | Команда клієнта (PHP) |
| :--- | :--- |
| **Створити** | `create|TYPE|ROOM|NAME` |
| **Видалити** | `delete|ID|` |
| **Перемкнути** | `switch|ID|on` (або `off`) |
| **Значення** | `set|ID|VALUE` |
| **Статус** | `status||` |
